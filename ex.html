Jekyll - a Ruby based site generator

Installation

  • Installed as a Ruby gem using a package manager (such as Homebrew)
  • (Install command line tools in Xcode)
  • $ gem install jekyll
  • Jekyll install instructions

    • Implementation

    • Jekyll is run through the command line
    • Enter a directory
    • $ cd Desktop/intentionjs
    • intentionjs$ Jekyll --server
    • The site will be available on Localhost:4000

    Organization

    • Jekyll suggests it be organized with specific organization model
    • However, depending on what the site is used for (blog or static) the configuation can be edited. These are the main layout tools:
    • _includes
    • _layouts
    • _posts (only if you are running as a blog)
    • _assets
    • _config.yml (essential for deploying to a url)

    Templating

    • Jekyll runs using Liquid templating language
    • All layout templates go in the _layout folder (duh)
    • Each file needs to call back to a layout using YAML front matter
    • ---
      Layout: post
      Title: Jekyll example
      ---
    • content is called by using tags such as {{ content }} in layout files
    • Liquid templating junk

    Publishing

    • Jekyll is an ideal mate for Github Pages
    • publishing to a github page is also done in the command line with git
    • $ git add .
      $ git commit -m "Add new content"
      $ git push origin master
    • A GitHub post-commit hook will automatically deploy your changes to your hosted blog.
